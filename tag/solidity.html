4 创建user.sol文件

使用solidity编写智能合约。 solidity是用于编写以太坊智能合约的语言。

例如：

如果要在区块链上进行“出勤登记”，你需要一个user对象数组。 它包含：
1. 访问用户的getter方法； 2. 设置用户的setter方法； 3. 验证用户是否需要登记出勤（因为每个用户每天只能出勤一次，后续触达时，需要判断一下是否需要登记出勤）

4.1 声明solidity编译器的版本

  1. 先查看本地的solidity版本是多少
    $ solc --version
  2. 在代码中声明编译器的版本 pragma solidity ^0.4.21

0.4.21是solidity的版本号

4.2 导入字符串处理库

import "stringUtils.sol"

4.3 为user定义合约

contract userRecords {}

4.4 初始化

为了存储user的name和gender信息，用enum类型定义性别类型，用struct定义用户

// 定义枚举类型的变量，用于存储用户性别
enum genderType { male, female }

// 储存在以太坊合约中的实际用户对象
struct user {
    string name;
    genderType gender;
}

4.5 声明用户对象

user user_obj;

4.6 添加getter，setter方法

我们需要在区块链上保存每个用户的信息。
将上述方法设置为public（因为要从外部访问）

// 设置用户的公共方法
function setUser(string name, string gender) public {
    genderType gender_type = getGenderFromString(gender);
    user_obj = user({name:name, gender: gender_type});
}
// 获取用户的公共方法
function getUser() public returns (string, string) { 
    return (user_obj.name, getGenderToString(user_obj.gender));
}

4.7 添加getGenderFromString方法和getGenderToString方法

// 将 字符串 转换为 genderType的枚举类型
function getGenderFromString(string gender) internal returns(genderType) {
    if(StringUtils.equal(gender, "male")) {
        return genderType.male;
    } else {
        return genderType.female;
    }
}
// 将 genderType的枚举类型 转换为 字符串
function getGenderToString(genderType gender) internal returns (string) {
    if(gender == genderType.male) {
        return "male";
    } else {
        return "female";
    }
}

4.8 完整示例

pragma solidity ^0.7.4;

// 导入字符串处理库
import "stringUtils.sol";

// 为user定义合约
contract userRecords {
  // 定义枚举类型的变量，用于存储用户性别
  enum genderType { male, female };
  // 储存在以太坊合约中的实际用户对象
  struct user{
    string name;
    genderType gender;
  }

  // 定义枚举类型的变量，用于存储用户性别
    enum genderType { male, female }

    // 储存在以太坊合约中的实际用户对象
    struct user {
        string name;
        genderType gender;
    }

  user user_obj;

  // 将 字符串 转换为 genderType的枚举类型
  function getGenderFromString(string gender) internal returns(genderType) {
    if(StringUtils.equal(gender, "male")) {
            return genderType.male;
        } else {
            return genderType.female;
        }
    }
  // 将 genderType的枚举类型 转换为 字符串
  function getGenderToString(genderType gender) internal returns (string) {
    if(gender == genderType.male) {
        return "male";
    } else {
        return "female";
    }
}

  // set user public function
  // This is similar to persisting object in db.
  function setUser(string name, string gender) public {
    genderType gender_type = getGenderFromString(gender);
    user_obj = user({name:name, gender: gender_type});
  }

  // get user public function
  // This is similar to getting object from db.
  function getUser() public returns (string, string) {
    return (user_obj.name, getGenderToString(user_obj.gender));
  }
}

5 使用python库进行编译/部署

5.1 实例化web3对象

web3对象用于和合约做交互，如，通过abi和contract_address获取合约中的user对象

from web3 import Web3
# web3.py instance
w3 = Web3(Web3.HTTPProvider("http://127.0.0.1:8545"))

5.2 使用python的solc编译器模块包，编译solidity代码

from solc import compile_files
# 编译所有的合约文件
contracts = compile_files(['user.sol', 'stringUtils.sol'])
# 分离 主文件 和 链接文件 的合约
main_contract = contracts.pop("user.sol:userRecords")
library_link = contracts.pop("stringUtils.sol:StringUtils")

5.3 连接 主文件合约 和 外链合约

编译主文件合约时，需要将 外链合约的部署地址 与 主文件合约 链接起来。
所以，先通过编译部署所有链接， 我们应该在签订合同之前部署库地址来替换这个部分

将主文件合约和外链库地址连接起来

from solc import link_code


def deploy_contract(contract_interface):
    # 根据byte和abi获取合约对象
    contract = w3.eth.contract(abi=contract_interface['abi'], bytecode=contract_interface['bin'])

    # 发布合约，获取交易hash
    tx_hash = contract.deploy(transaction={'from': w3.eth.accounts[1]})

    # 获取hash收据，用于获取合同地址
    tx_receipt = w3.eth.getTransactionReceipt(tx_hash)

    return tx_receipt['contractAddress']


library_address = {
    'stringUtils.sol:StringUtils': deploy_contract(library_link)
}
main_contract['bin'] = link_code(main_contract['bin'], library_address)

执行上述python代码，会发现地址已经连接起来了

5.4 利用w3部署合约

contract_address = deploy_contract(main_contract)

以上，一个智能合约就部署成功了。但我们需要在abi中触发合约，所以需要将这些信息保存起来

5.5 将编译后的内容写入文件compile.json

with open('compile.json', 'w') as outfile:
    data = {
        'abi': main_contract['abi']
        'contract_address': deploy_contract(main_contract)
    }
    json.dump(data, outfile, indent=4, sort_keys=True)

5.6 完整示例

from web3 import Web3
from solc import compile_files
from solc import link_code


w3 = Web3(Web3.HTTPProvider('http://127.0.0.1:8545'))

# 编译所有合约文件
contracts = compile_files(['user.sol', 'stringUtils.sol'])

# 分离 主文件 和 外链文件
main_contract = contracts.pop('user.sol:userRecords')
library_link = contracts.pop('stringUtils')


def deploy_contract(contract_interface):
    # 初始化并且部署合约
    contract = w3.eth.contract(abi=contract_interface['abi'], bytecode=contract_interface['bin'])

    # 发布合约，获取交易hash
    tx_hash = contract.deploy(transaction={'from': w3.eth.accounts[1]})

    # 获取hash收据，用于获取合同地址
    tx_receipt = w3.eth.getTransactionReceipt(tx_hash)

    return tx_receipt['contractAddress']


library_address = {
    'stringUtils.sol:StringUtils': deploy_contract(library_link)
}
main_contract['bin'] = link_code(main_contract['bin'], library_address)


with open('compile.json', 'w') as outfile:
    data = {
        'abi': main_contract['abi']
        'contract_address': deploy_contract(main_contract)
    }
    json.dump(data, outfile, indent=4, sort_keys=True)

6 FlaskAPI

合约部署成功了（仅部署一次即可），然后可以通过flaskapi来做数据 “存”，“取” 操作。

6.1 创建post方式的api来获取用户信息

from flask import Flask, Response, request, jsonify
from marshmallow import Schema, fields, ValidationError


def check_gender(data):
    valid_list = ['male', 'female']
    if data not in valid_list:
        raise ValidationError('Invalid gender. Valid choices are' + valid_list)


class UserSchema(Schema):
    name = fields.String(required=True)
    gender = fields.String(required=True, validate=check_gender)


app = Flask(__name__)


@app.route('/blockchain/user', methods=['POST'])
def user():
    body = request.get_json()
    res, err = UserSchema().load(body)
    if err:
        return jsonify(err), 422
    return jsonify({'data': res}), 200

6.6 完整示例

import json
from flask import Flask, Response, request, jsonify
from marshmallow import Schema, fields, ValidationError
from web3 import Web3


# 获取web3实例
w3 = Web3(Web3.HTTPProvider("http://127.0.0.1:8545"))
w3.eth.defaultAccount = w3.eth.accounts[1]

# 从已经编译后，生成的json文件中获取abi和合约地址
with open('compile.json', 'r') as f:
    datastore = json.load(f)
    abi = datastore['abi']
    contract_address = datastore['contract_address']


# 检测性别合法性
def check_gender(data):
    valid_list = ['male', 'female']
    if data not in valid_list:
        raise ValidationError('Invalid gender. Valid choices are' + valid_list)


# 定义合约中的user模型
class UserSchema(Schema):
    name = fields.String(required=True)
    gender = fields.String(required=True, validate=check_gender)


app = Flask(__name__)


@app.route('/blockchain/user', methods=['POST'])
def user():
    # 通过abi和合约地址获取合约对象
    user = w3.eth.contract(address=contract_address, abi=abi)

    # 从http的request对象中获取数据
    body = request.get_json()
    res, err = UserSchema().load(body)
    if err:
        return jsonify(err), 422

    # 通过合约对象可以调用合约中的任何public方法
    # 调用serUser方法，修改user内容
    tx_hash = user.functions.setUser(res['name'], res['gender'])

    # 通过transact使用户信息public。即用户信息上链
    tx_hash = tx_hash.transact()

    # 等待交易被打包
    w3.eth.waitForTransactionReceipt(tx_hash)

    # 通过call方法获取修改后的用户信息
    user_data = user.functions.getUser().call()

    return jsonify({'data': user_data}), 200
